# php에서 변수명 앞에 _를 붙이는 이유? eg) _csrf
    - 내부용(Private) 이거나 외부에서 직접 접근하지 말라는 표시
    - 클래스 속성에 _token을 붙여 로컬 변수와 구분하기 위해
    - 일시적이거나 특별한 목적의 변수
        - foreach ($items as $_item) { ... $_item은 루프 안에서만 잠깐 쓰임
    - 일부 프레임워크(예: Laravel, Symfony, CakePHP)에서 내부적으로 쓰는 변수나 헬퍼 메서드에 _를 붙이기도함
    - PHP 슈퍼 글로벌 변수명들: $_SERVER, $_POST, $_GET, ...
